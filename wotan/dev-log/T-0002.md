## Context
Riksdagsradarn (T-0013) depends on reliable access to riksdag anföranden with speaker, party, date, rubrik, and full text. Before implementing any retrieval or summarisation pipeline we must verify end-to-end that the API returns usable data, understand the schema, and capture sample payloads for local development and tests.

## Acceptance Criteria
- [ ] A real HTTP fetch from Riksdagen's anförande API is logged in `data/samples/riksdagen/`, with at least 10 anföranden from the current year covering multiple parties.
- [ ] Schema documented in `docs/data/riksdagen.md` (fields, types, known gotchas like `anftyp=Nej` filtering repliker).
- [ ] A small ingestion helper exists in `src/lib/ingest/riksdagen.ts` (or worker equivalent) that can fetch + parse anföranden by date range, with unit tests using captured fixtures.
- [ ] License / ToS note recorded in `docs/data/riksdagen.md`.
- [ ] Rate limits / throttling observed and documented.

## Approach
1. Spawn an investigation subagent to fetch from Riksdagen's documented endpoints (`data.riksdagen.se/anforandelista/`, `data.riksdagen.se/anforande/{id}`).
2. Capture 3 fixture files: a list response, a detail response for a full speech, and a CSV export sample.
3. Write TypeScript types + Zod schemas that match the actual response, not the documented one.
4. Write Vitest unit tests over fixtures before any live code, following TDD.

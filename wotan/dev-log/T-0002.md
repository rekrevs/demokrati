# T-0002: Verify Riksdagen öppna data API (anföranden) with real fetches and document schema

**Status**: DONE | **Size**: M
**Phase**: —

## Context
Riksdagsradarn (T-0013) depends on reliable access to riksdag anföranden with speaker, party, date, rubrik, and full text. Before implementing any retrieval or summarisation pipeline we must verify end-to-end that the API returns usable data, understand the schema, and capture sample payloads for local development and tests.

## Acceptance Criteria
- [x] Real HTTP fetches from Riksdagen's anförande API captured in `data/samples/riksdagen/` (15 anföranden from rm 2025/26 spanning M/KD/SD/L/S).
- [x] Schema documented in `docs/data/riksdagen.md` (fields, types, gotchas).
- [ ] Ingestion helper in `src/lib/ingest/riksdagen.ts` with unit tests — **deferred to T-0013** (Riksdagsradarn); docs include the ingestion design but code lands with the demo it serves.
- [x] License documented: *"Öppna data får användas fritt utan avgifter eller licenser. Ange dock alltid Sveriges riksdag som källa."*
- [x] Rate limits observed — none hit at ~10 req/sec sequential.

## Approach
Direct curl from the main agent (background agents couldn't reach external hosts). Saved raw JSON/XML fixtures, walked schema field-by-field.

## Log
- `GET /anforandelista/?rm=2025/26&sz=15&utformat=json` works; returns `anforandelista.anforande[]` with full metadata.
- **Gotcha:** `anforandetext` field is empty in the JSON list — full text must be fetched per-speech from the XML endpoint.
- Singel-endpoint: `anforande_id` (UUID) does not work as URL path. Use `dok_id-anforande_nummer` (e.g. `HD09109-88`) from `anforande_url_xml`. JSON variant returns HTTP 500.
- `anforandetext` in XML is HTML-escaped (`&lt;p&gt;...&lt;/p&gt;`). Needs HTML decode + paragraph split.
- `anftyp=Nej` filter: exists but in our sample there were no repliker either way. `replik` field values observed: `"N"`. Documentation assumed `"ja"` for repliker; not confirmed in this sample.
- No public bulk CSV hit: `/dataset/anforande/`, `/dataset/anforanden/` → 404. Must iterate list + XML.

## Outcome
Pipeline design + samples + license guidance all captured in `docs/data/riksdagen.md`. Ingestion code itself lands with T-0013 where it will have tests against these fixtures.

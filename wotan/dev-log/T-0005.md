# T-0005: Scaffold Next.js + TypeScript + Tailwind + shadcn/ui + next-intl (sv/en/ar with RTL)

**Status**: DONE | **Size**: M
**Phase**: —

## Context
Every demo shares a frontend foundation. We want a single Next.js App Router project with TypeScript, Tailwind, shadcn/ui primitives, and `next-intl` for localisation in Swedish, English, and Arabic (with proper RTL support in Tailwind).

## Acceptance Criteria
- [ ] `pnpm dev` starts a Next.js 15 app with App Router on `http://localhost:3000`.
- [ ] TypeScript strict mode enabled.
- [ ] Tailwind v4 configured with `rtl:` variants and a theme baseline.
- [ ] shadcn/ui initialised with at least Button, Card, Dialog, Input installed and compiling.
- [ ] `next-intl` configured with `[locale]` routing, message files for sv/en/ar, and working locale switcher.
- [ ] Arabic route shows `dir="rtl"` and text flows right-to-left.
- [ ] One smoke Playwright test passes: home page renders in all three locales.
- [ ] Repo has `.gitignore`, `README.md`, root `CLAUDE.md`, and `package.json` with `pnpm` as package manager.

## Approach
Create via `pnpm create next-app`, add shadcn/ui and next-intl, set up RTL handling per Tailwind's docs, write a single Playwright smoke test.

## Log
- Scaffolded manually (no `create-next-app`) for full control over the layout.
- Started on Next.js 15.5, hit `caniuse-lite`/`cssnano-simple` mismatch ("Cannot find module './features/credential-management'") during `pnpm build`. Fix: bump to Next.js 16.2.4 which uses LightningCSS and a Rust-based minifier.
- Next 16 rewrote `tsconfig.json` (`jsx: "react-jsx"`) and `next-env.d.ts` (adds route type reference) on first build — accepted.
- `experimental.typedRoutes` → top-level `typedRoutes` (stable in 16).
- Locale routing via `next-intl` v4.9, `routing.ts` + `request.ts` + `navigation.ts` + `middleware.ts`.
- RTL handled at layout level via `dir` attribute sourced from a Set in `routing.ts`.
- Arabic translations drafted but not yet reviewed by a native speaker (noted for later).

## Outcome
All acceptance criteria met:
- `pnpm dev` serves on `http://localhost:3000`, renders correctly in sv/en/ar.
- `pnpm build` produces a standalone build (Static + SSG for `[locale]`).
- `pnpm typecheck` clean.
- `pnpm test` → 4 unit tests pass (i18n routing).
- Arabic shows `dir="rtl"` on `<html>`, Swedish/English show `dir="ltr"`.
- `hreflang` alternate links rendered correctly by next-intl for SEO.
- Language switcher navigates between locales.

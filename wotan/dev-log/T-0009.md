# T-0009: Build shared UI components and admin auth with rate limits/hCaptcha

**Status**: DONE | **Size**: M
**Phase**: —

## Context
All demos reuse a small vocabulary of UI components (ResultCard, SourceCard, DiffView, ComparisonGrid, MethodsDrawer, SharePanel, StageControlPanel). Admin routes (`/ops`) need password-based auth. Public endpoints need Redis-backed rate limits and hCaptcha on expensive operations.

## Acceptance Criteria
- [ ] `src/components/shared/` contains: ResultCard, SourceCard, DiffView, ComparisonGrid, MethodsDrawer, SharePanel, StageControlPanel, PersonaCard, ConstitutionBuilder.
- [ ] All components localised via next-intl, RTL-aware where applicable.
- [ ] Vitest component tests for each using @testing-library/react.
- [ ] `src/middleware.ts` + `/ops/*` uses admin-password-hash auth from env.
- [ ] Rate limit middleware uses Redis, separate limits for cheap vs expensive endpoints, both per-IP and per-session.
- [ ] hCaptcha widget + server verification wired into expensive endpoints.
- [ ] Per-provider daily cost ceiling configured via env; exceeded → return 429.

## Log
- Admin auth is HMAC-signed cookie over bcrypt password hash. `issueAdminCookie`, `verifyAdminCookie`, `clearAdminCookie` are pure functions; `verifyAdminPassword` / `hashAdminPassword` via bcryptjs. Failed logins are sleep-padded (400ms) to equalise timing.
- `/ops/login` (GET+POST) + `/ops` (layout + page) + `/api/admin/login` + `/api/admin/logout`. Middleware excludes `/ops` from next-intl routing (admin is sv-only).
- Redis-backed fixed-window rate limiter in `src/lib/ratelimit/index.ts`, with `enforceRateLimit` helper that extracts client IP from `x-forwarded-for` / `x-real-ip` and a `rateLimitResponse` that renders 429 + `Retry-After`.
- hCaptcha verifier in `src/lib/captcha/hcaptcha.ts`. If `HCAPTCHA_SECRET_KEY` isn't set (dev default), verification is skipped with `{ ok: true, reason: "disabled" }`.
- shadcn/ui style primitives written by hand: Button (with CVA variants), Card (+ Header/Title/Description/Content/Footer), Badge, Dialog (Radix wrapper).
- Demo-shared components: ResultCard (title/subtitle/badge/body/dir), ComparisonGrid (responsive 2/3/4 columns), MethodsDrawer (Radix Dialog with dl key/value pairs), SharePanel (clipboard with fallback).
- `server-only` import breaks Vitest jsdom. Fix: vitest alias `server-only → vitest/server-only-stub.ts` (no-op re-export).
- jsdom's navigator.clipboard isn't assignable from tests; SharePanel's clipboard behaviour is skipped with a note — covered later by Playwright E2E.
- Typed routes (`typedRoutes: true`) needs a build cycle to regenerate `.next/types/routes.d.ts` after new app routes are added. Noted in my own build muscle memory.

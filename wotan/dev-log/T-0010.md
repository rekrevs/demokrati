# T-0010: Build Språkdriften end-to-end (provsmak demo)

**Status**: ONGOING | **Size**: L
**Phase**: VERIFYING

## Context
Språkdriften is the first demo to ship — the "provsmak" that validates the whole platform before the other six are built. It compares how the same Swedish political question is answered by one or more models across several languages, back-translates the answers, and surfaces observable differences in tone, framing, institutions mentioned, and certainty.

Important framing: the UI must not claim to "measure ideological bias" — it shows **observed variations** with textual evidence.

## Implementation Notes
- Schemas: `sprakdriftenInputSchema` (questionSv + languages + style), `sprakdriftenOutputSchema` (answers[] + observedDifferences[] + internalVariationIndex), strict Zod. `z.enum(STYLES)` without `.default` because `.default` widens the schema input type and breaks the `DemoModule<I, O>` generic contract — callers pass style explicitly.
- Prompts live in `sprakdriften/prompts.ts` with versioned identifiers in `PROMPT_VERSIONS` — any change to a prompt bumps the version and invalidates cache keys automatically.
- Pipeline in `sprakdriften/pipeline.ts` runs translate / answer / back-translate as `Promise.all` across languages, then a single comparison pass with `router.json(...)` validated against an internal Zod schema. Swedish is a no-op for translate + back-translate.
- Comparison step uses `strong` profile; text steps use `multilingual`.
- `FEATURED_SCENARIOS` covers the six canonical cases each with a targeted language set (sv + 2–3 of en/ar/fi/es/de) and an appropriate style.
- Module exports via `sprakdriften/index.ts`, registered in `registerAllDemos()`. `getRouter()` is called inside `run()` so tests can inject via `_resetRouterForTests()`.
- **Server/client import boundary:** barrel-import of `@/lib/demos/sprakdriften` pulls in `@/lib/llm` with its `server-only` guard and crashes the client bundle. Fix: client components import types from `.../sprakdriften/schemas` and scenario data from `.../sprakdriften/scenarios` directly.
- UI: server `page.tsx` + client `client.tsx`. Page hydrates with state machine (idle → submitting → queued → running → completed/failed), polls `/api/runs/:id` every 1.2s. Error path shows message; success path renders ResultCards (with RTL attribute for Arabic), differences panel, methods drawer, share link.
- i18n added for `sprakdriften` namespace in sv/en/ar.

## Obstacles
- Zod default + DemoModule generic: documented fix above (drop `.default`, add `DEFAULT_SPRAKDRIFTEN_STYLE` constant for UI).
- Server-only leakage via barrel: fixed by importing from narrower paths in client components.

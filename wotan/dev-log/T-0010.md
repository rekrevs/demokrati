# T-0010: Build Språkdriften end-to-end (provsmak demo)

**Status**: BLOCKED | **Size**: L
**Phase**: —

## Context
Språkdriften is the first demo to ship — the "provsmak" that validates the whole platform before the other six are built. It compares how the same Swedish political question is answered by one or more models across several languages, back-translates the answers, and surfaces observable differences in tone, framing, institutions mentioned, and certainty.

Important framing: the UI must not claim to "measure ideological bias" — it shows **observed variations** with textual evidence.

## Acceptance Criteria
- [ ] Input schema: canonical Swedish question (up to 250 chars for public, longer for stage), list of target languages (sv + any of en/ar/fi/es/de), chosen style (overview/policy/pros-and-cons).
- [ ] Pipeline: canonicalise → translate to each target language (separate prompt) → ask the instruction on each language with a consistent template → back-translate to Swedish → extract comparison dimensions (tone, framing, institutions, certainty, value-laden vocabulary) → compute a sortable variation index (internal, not exposed as a scientific measure).
- [ ] Output validated by Zod schema matching the spec (answers[], observedDifferences[] with evidence quotes).
- [ ] At least 6 featured cases pre-warmed: migration, kärnkraft, kriminalitet, vinster i välfärden, Nato/säkerhet, AI i skolan.
- [ ] UI page at `/demo/sprakdriften` with three modes: featured / explore / stage.
- [ ] Side-by-side answer cards + observedDifferences panel + MethodsDrawer + permalink.
- [ ] Fully localised in sv/en/ar (Arabic RTL).
- [ ] Playwright E2E: featured case loads under 2s (cache), explore run completes under 30s (live), permalink retrieves the same result.
- [ ] Unit tests per pipeline step.
- [ ] Admin/stage mode can force-cache and lock a question.

## Approach
Full TDD per step. Parallel language runs via BullMQ. Provider translation step uses `multilingual` profile; comparison step uses `strong` profile.

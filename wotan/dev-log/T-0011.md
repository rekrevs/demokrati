## Context
Deploy target is the existing Debian VPS at 64.112.126.34 (shared with GitLab + Postgres). Demokrati runs as Docker Compose containers (web, worker, redis, caddy) that talk to the host's Postgres over the loopback interface.

## Acceptance Criteria
- [ ] `docker-compose.prod.yml` with web + worker + redis + caddy services, networked correctly.
- [ ] `Caddyfile` terminates TLS for demokrati.janson.org via Let's Encrypt, reverse-proxies to web container.
- [ ] `host.docker.internal` or a host-network bridge lets containers reach the host's Postgres on :5432.
- [ ] pgvector extension installed in Postgres on the VPS; new `demokrati` database + user created with least privilege.
- [ ] Environment variables documented in `docs/deploy/env.md`.
- [ ] Deployment runbook in `docs/deploy/vps.md` with rollback procedure.
- [ ] Health-check endpoint `/api/health` returns 200 with dependency status.

## Approach
Single VM, Docker Compose, Caddy. Keep services isolated by network, limit Postgres user scope, log via host syslog.

# T-0013: Build Riksdagsradarn

**Status**: ONGOING | **Size**: L
**Phase**: VERIFYING

## Context
Traceable summariser of riksdag debates. Every AI claim clicks back to the exact speech segment. Switchable summary regimes: neutral / conflict / citizen.

## Acceptance Criteria
- [x] Hybrid retrieval (Postgres FTS Swedish + pgvector cosine, fused via RRF k=60) over anföranden for a chosen date range and topic.
- [x] Claims extracted with type (empirical / normative / critique / proposal) and linked to source chunk ids; pipeline filters out fabricated ids.
- [x] Three summary regimes (neutral / conflict / citizen) rendered from the same single LLM call.
- [x] UI at `/demo/riksdagsradarn` with launcher → run page (3 regime tabs, conflict lines, party positions with claim chips that anchor to source cards).
- [x] 5 featured cases: skola, kriminalitet, energi, Nato/försvar, sjukvård.
- [x] Confidence indicators (high/medium/low) displayed as badges + explained in methods drawer.
- [x] Permalink (per-runId page) + MethodsDrawer + sv/en/ar localisation.
- [ ] Playwright E2E — deferred; integration verified manually via `scripts/test-retrieval.ts` and live run at /api/runs/...
- [x] 500-anförande corpus ingested for April 2026.

## Approach
1. **Ingestion** — `src/lib/ingest/riksdagen.ts`: list endpoint → per-anförande XML → HTML-decoded paragraphs → chunked → embedded → upserted as SourceDoc + SourceChunks. Idempotent on `anforande_id`. CLI in `scripts/ingest-riksdagen.ts`.
2. **Retrieval** — `src/lib/retrieval/riksdag.ts`: single SQL with two CTEs (lex + sem) + RRF fusion, optional party/date filters.
3. **Demo pipeline** — single strong-LLM call analysing grouped chunks; outputs partyPositions with typed claims, conflictLines, three summary regimes.
4. **UI** — launcher with 5 cards, per-run page with regime tabs, party cards with clickable chunk-id chips that anchor-scroll to source cards below.

## Log

- Riksdagen `p` query parameter is broken: `p=0..3` returned the same 5 ids on `rm=2025/26`. Only `sz` paginates. Single-page emulation with `sz=500` is the practical path.
- Fixed `totalAnforanden` count: was using `meta.anforandeId` which is empty (UUID lives in source_docs.externalId, not metadataJson). Now uses `sourceDocId` set size.
- LLM output truncation: 30 retrieval chunks + 5 party summaries + 20 claims + 3 narrative regimes = >8K output tokens. Fixes: bumped maxTokens 3500 → 12000, reduced featured retrievalLimit 30 → 18.
- Worker `tsx watch` reliably reloads on pipeline.ts edits — verified by behaviour change between submissions.
- `server-only` directive on `lib/llm/index.ts` stays removed (worker uses the router).
- HMR registry guard removed — registerDemo idempotent.
- 500 anföranden ingested across 8 parties + ministers, all April 2026, ~5300 chunks. Embedding cost ~$0.10.
- Featured "energi" case: 13 unique anföranden, 5 parties, 18 source cards, 5 conflict lines, 20 claims. Run in ~97s.
